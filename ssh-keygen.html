WHAT IS AN SSH KEY?
// OVERVIEW
An SSH key is a cryptographic key pair used to authenticate securely without a password. It consists of a private key (kept secret on your machine) and a public key (shared with servers / GitHub / etc.). The algorithm recommended today is Ed25519 — fast, secure, and compact.
EXECUTE_PROTOCOL
GENERATE YOUR KEY
  1. OPEN TERMINAL
    Launch your terminal application. On macOS use Terminal or iTerm2. On Linux use your distro's terminal emulator.
  2. RUN KEYGEN COMMAND
    Generate an Ed25519 key pair. Replace the email with your own.
    BASH
    $ ssh-keygen -t ed25519 -C "you@example.com"
    // NOTE
    If your system does not support Ed25519, use -t rsa -b 4096 as a fallback.
  3. CHOOSE FILE LOCATION
    The default path is fine for most users. Press Enter to accept it.
    PROMPT
    Enter file in which to save the key (~/.ssh/id_ed25519): # press Enter to use the default location
  4. SET A PASSPHRASE
    Enter a strong passphrase (recommended) or press Enter twice to skip. A passphrase encrypts your private key on disk.
    // TIP
    Use ssh-agent to cache your passphrase so you don't have to type it every time.
  5. VIEW YOUR PUBLIC KEY
    Copy the public key — this is what you add to GitHub, servers, etc.
    BASH
    $ cat ~/.ssh/id_ed25519.pub
    // EXAMPLE OUTPUT
    ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAI... you@example.com
  6. COPY KEY TO CLIPBOARD
    Use the appropriate command for your OS to copy the key to your clipboard.
    MACOS
    $ pbcopy < ~/.ssh/id_ed25519.pub # macOS clipboard
    LINUX
    $ xclip -selection clipboard < ~/.ssh/id_ed25519.pub # Linux (requires xclip)
  7. ADD TO SSH AGENT (OPTIONAL)
    Start the agent and add your key so you do not need to enter the passphrase repeatedly.
    BASH
    $ eval "$(ssh-agent -s)" # start the agent
    $ ssh-add ~/.ssh/id_ed25519 # add your key
  1. OPEN POWERSHELL OR CMD
    Right-click the Start button and select Windows Terminal, PowerShell, or Command Prompt. OpenSSH ships with Windows 10 (1809+) by default.
    // CHECK OPENSSH
    Run ssh -V to confirm OpenSSH is installed. If not, install it via Settings → Apps → Optional Features → OpenSSH Client.
  2. RUN KEYGEN COMMAND
    The same command works in PowerShell and CMD on Windows.
    POWERSHELL
    PS> ssh-keygen -t ed25519 -C "you@example.com"
  3. ACCEPT DEFAULT LOCATION
    Press Enter to save the key to the default location: C:\Users\YourName\.ssh\id_ed25519
  4. SET A PASSPHRASE
    Enter a passphrase (recommended) or press Enter twice to skip.
  5. COPY YOUR PUBLIC KEY
    Use PowerShell to copy the key to your clipboard.
    POWERSHELL
    PS> Get-Content $env:USERPROFILE\.ssh\id_ed25519.pub | Set-Clipboard
  6. ADD TO SSH AGENT (OPTIONAL)
    Start the OpenSSH agent service and add your key.
    POWERSHELL (ADMIN)
    PS> Set-Service -Name ssh-agent -StartupType Automatic
    PS> Start-Service ssh-agent
    PS> ssh-add $env:USERPROFILE\.ssh\id_ed25519
    // NOTE
    The Set-Service command requires an elevated (Administrator) PowerShell window.
DEPLOY_TARGETS
WHERE TO ADD YOUR PUBLIC KEY
// GITHUB
Settings → SSH and GPG keys → New SSH key → paste your public key.
// REMOTE SERVER
Append your public key to ~/.ssh/authorized_keys on the remote host, or use:
BASH
$ ssh-copy-id user@your-server.com # copies your public key to the remote server automatically
SECURITY_PROTOCOL
SECURITY NOTES
// WARNING
NEVER share your private key (~/.ssh/id_ed25519). Only the .pub file is meant to be shared.
// BEST PRACTICES
Use a unique key per device. Rotate keys periodically. Always set a passphrase. Keep your private key file permissions at 600 (chmod 600 ~/.ssh/id_ed25519).
